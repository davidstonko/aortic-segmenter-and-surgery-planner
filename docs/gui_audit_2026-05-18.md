# GUI usability audit ŌĆö 2026-05-18

Audit of `+app/AorticCenterlineApp.m` against general UI best practices
(hit-targets, spacing, accessibility, responsive layout, empty/error
states, discoverability). Screenshots captured at three window sizes
into `results/figures/gui_audit/`.

## Method

1. Launched the app fresh; navigated each of the 6 steps via
   `setStepPublic` and captured a screenshot.
2. Re-rendered Step 3 at 1280├Ś720 (laptop) and 1100├Ś700 (documented
   minimum) to verify layout adapts.
3. Grep-audited `+app/AorticCenterlineApp.m` for layout primitives.
4. Read the entry points to each step's side-panel builder.

### Ō£ģ Things the GUI does well

- Side-panel content area is `Scrollable='on'` ŌĆö vertical overflow handled.
- Mode toggle (User-driven / Automatic) renders consistently on every step.
- Ōōś info buttons on every section + every key control (good discoverability).
- Top-level `Help` menu with structured entries (pipeline overview, glossary, etc.).
- Color coding by functional group (green = measurement, orange = W/L, slate = view).
- Step bar visually shows progress (active step blue, done green).
- Window centers on screen and respects min/max dimensions.

## Recommended next steps (prioritised)

1. **[P0]** Add `figureSizeChangedFcn` that repositions the StepBar / ViewToolbar / ToolToolbar / ImagePanel / SidePanel based on current `UIFigure.Position`. Without this, the GUI breaks on any user resize. (~2 hours)
2. **[P1]** Convert the step bar from `uilabel` to `uibutton`s with a proper hit-target so users discover that steps are clickable.
3. **[P1]** Add `Enable` gating to tool toolbar buttons that require a volume / mask / centerline.
4. **[P2]** Migrate the major containers to `uigridlayout` so future-resize-handling is automatic. Migrate one panel at a time; tests should keep passing.
5. **[P2]** Move the floating left overlay buttons into the tool toolbar (or a left-edge rail) so they don't occlude the image area.
6. **[P2]** Add tooltips to every uncovered button.
7. **[P3]** Consider an onboarding center-panel on Step 1 with "Open DICOM / Open phantom / Recent files" cards.
