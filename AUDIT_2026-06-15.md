## A. Critical & High findings

### 🔴 CRITICAL

**A0 — repro-1 · Real PHI embedded in `.mat` artifacts that aren't git-ignored.**
`results/logs/johndoe2_ct.mat`, `results/logs/ct_volume.mat`, `results/logs/johndoe1_after_cfa_extend.mat`, `results/logs/johndoe2_diag_adaptive/stage_masks.mat` (and 2 files under the git-ignored `.cache/planner_result/`) embed the full DICOM `D.info_first` struct with **PatientName, PatientID/MRN, DOB, AccessionNumber, InstitutionName, ReferringPhysicianName** (real values for both cases — redacted here). The two project case labels are the patients' real surnames. `.gitignore` covers `results/cache/` + `results/figures/` but **NOT `results/logs/`**, so a `git add -A` would publish them. *No live breach yet (tree isn't a committed git repo), but must be purged before any publish.*
**Fix:** (1) add `results/logs/`, `results/snapshots/`, `results/videos/`, `results/**/*.mat` to `.gitignore` now; (2) strip `info_first` PHI from the cached structs (keep only de-identified geometry/spacing) or regenerate via `preprocess.anonymize_dicom_dir`; (3) grep the whole tree for the patient surnames before any commit; (4) reconsider the "JohnDoe1"/"JohnDoe2" labels themselves.

### 🟠 HIGH

**A1 — sizing-1 · Neck diameter over-called ~30% on aneurysm-present cases.** `+evar_plan/measure_from_centerline.m:188-210`. The neck baseline is anchored to `neck_min_idx = aneurysm_idx-1` (the *widest* non-aneurysmal node), and the proximal-walk threshold `R_neck_baseline*1.20` rarely fires, so `seal_start_idx` collapses to the supraceliac-skip point and `neck_diameter_mm` averages `R` from the renal level *through the dilating segment*. On the project's own test profile (8 mm neck ramping 8→18) this yields ≈21 mm vs a true ≈16 mm. Graft oversizing and IFU diameter-eligibility are computed off this. **Pre-existing — not introduced by the B1–B3 work, which only touched angles/labels.** *[NEEDS DECISION on seal-zone convention.]*

**A2 — sizing-2 · Neck diameter mean includes the aneurysmal node.** Same file, `:210` averages `R(seal_start_idx:aneurysm_idx)` *inclusive* of `aneurysm_idx`, which is by definition the first node with `R > aneurysm_R_mm`. Biases every aneurysm-case neck Ø upward; endpoint should be `aneurysm_idx-1`. Compounds A1.

**A3 — sizing-3 · Vacuous IFU eligibility on all-NaN measurements.** `+ifu/check_eligibility.m` skips each criterion when its measurement is NaN; a device with *all* measurements NaN accrues no fail reasons → `eligible = true`, and `match_devices`/`generate_plan` then print "Recommended device: X" with no anatomic basis. A failed centerline / short polyline produces a confident bogus recommendation. *[NEEDS DECISION on the minimum non-NaN criterion set.]*
**Fix:** require core criteria (neck Ø + both iliac Ø) non-NaN for `eligible=true`; otherwise mark "indeterminate"; surface a count of criteria actually evaluated.

**A4 — code-1 · Pipeline continues silently after a caught segmentation-extension failure.** `run_planner_headless.m:180-203` catches branch/CFA-extension errors, `fprintf`s, and proceeds to emit a full plan from the raw/truncated mask with no machine-readable flag. A clinician reading the headless plan has no signal the iliac/CFA reach was never extended.
**Fix:** record a `result.qc.segmentation_incomplete` flag + warning and surface it in the plan text and `planner_qc.png`.

**A5 — seg-1 · Cranial-first slice orientation is assumed, not verified.** `+autoseg/extend_and_detect_branches.m:185-404` hardcodes visceral search bands as fractions of aorta z-extent assuming Z=1=cranial. `dicom_load.m` only guarantees that when `ImagePositionPatient` exists; the `InstanceNumber` fallback does not. A feet-first / reversed-instance real series inverts every band and silently drops all visceral branches.
**Fix:** validate craniocaudal direction after the InstanceNumber fallback (from `ImageOrientationPatient` or a body-region heuristic) and flip; or assert an orientation flag on `D`.

**A6 — docs-1 · STATUS.md predates B1–B3 and carries stale test counts.** `STATUS.md:1-5,87` (as-of 2026-06-06) never mentions α/β / lumen / `aneurysm_detected` / N/A-neck, and claims "16 unittest classes (76 tests)" (actual: 26 classes / ~134 methods; non-GUI 110/111). It is the doc that HANDOFF/CONTRIBUTING point to as "live state."
**Fix:** add a top "Status (2026-06-15)" block summarizing B1–B3 + mesh + AortaSeg24 scaffold; correct the test-count line.

**A7 — tests-4 · Mesh export has zero automated coverage.** `+evar_plan/export_mesh.m` + GUI `saveMesh` + `exportEverythingAuto`'s `lumen.stl` (all added this session) are referenced by no test; only a one-off manual vert/face check exists. A `pixel_mm`-scaling or empty-mask regression would pass CI silently.
**Fix:** add a `test_export_mesh` (synthetic mask + known spacing → assert STL written, >0 verts/faces, extent matches mask scaled to mm).

**A8 — repro-2 · `.gitignore` gap is the mechanism behind A0.** `.gitignore:15-16` ignores only `results/cache/` + `results/figures/`. (Same fix as A0 part 1; listed separately because it's the one-line change that closes the leak vector.)

## B. Needs your decision (clinical correctness)

These change a number an operator reads/exports — the vascular-surgeon's call, not the auditor's.

- **B-i (A1/A2) — Seal-zone / neck-diameter convention.** How should the proximal neck diameter be defined and over what seal-zone length? Current code over-calls it. Recommendation: R-minimum baseline + a fixed ~15 mm seal zone just proximal to aneurysm onset, excluding the onset node. Your call on the seal-zone length and whether to use min vs mean.
- **B-ii (A3) — Minimum criterion set for IFU eligibility.** Which measurements must be present for a device to be called "eligible" vs "indeterminate"? Recommendation: neck Ø + both iliac Ø mandatory.
- **B-iii (sizing-4) — Repeatability tolerance.** α/β/neck-length pivot on threshold-triggered node indices; a resample/rotation can shift them. Do you want hysteresis on aneurysm-onset + a reported sensitivity band? Matters for a publishable reproducibility claim.
- **B-iv (seg-2) — Lumen-only ceiling.** Every diameter is lumen-based and under-calls the thrombus-laden outer-wall sac (the JohnDoe2 finding). Decide: add outer-wall/ILT segmentation, or explicitly scope the tool + benchmark to lumen measurements. (Ties to goal #26, which AortaSeg24 does **not** solve.)
